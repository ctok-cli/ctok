## A free Claude token estimator and prompt refiner

**ctok** is an open-source toolkit for everyone who builds on the Claude API.
It counts tokens accurately for **Claude Haiku 4.5**, **Sonnet 4.6**, and
**Opus 4.7**, calculates the dollar cost of your prompt at current Anthropic
list pricing, recommends the right model and effort level for the task, and
refines your prompt with a 7-pass heuristic pipeline that strips filler,
catches repeats, and adds structure.

It runs everywhere a developer already works: a CLI on the terminal, an MCP
server inside Claude Code / Cursor / Zed, a web playground in the browser,
plus extensions for VS Code, JetBrains, Slack, Discord, and a GitHub Action
that gates pull requests on token budgets.

## What ctok does

  <Card title="Estimate tokens" icon="magnifier">

  </Card>

  </Card>

## Quick install

```sh
# npm
npm install -g ctok

# curl (macOS / Linux)
curl -fsSL https://ctok-cli.github.io/ctok/install.sh | sh

# PowerShell (Windows)
irm https://ctok-cli.github.io/ctok/install.ps1 | iex

# Homebrew
brew tap ctok-cli/tap && brew install ctok

# Scoop
scoop bucket add ctok https://github.com/ctok-cli/scoop-ctok && scoop install ctok
```

## 30-second demo

```sh
$ ctok check "Refactor the auth middleware to use the new session store"

  Input tokens    ~367  (range 365-369, high confidence)
  Output tokens   ~400  (range 400-551)
  Estimated cost  $0.035  at Opus 4.7

  Recommendation  opus-4-7 · high effort
  Why             Deep / architectural work. Opus 4.7 has the strongest
                  reasoning and is worth the premium when correctness matters.

$ ctok refine "please can you kindly help me to handle the auth thing somehow"

  Score 12 → 45 (+33)  ·  saved 4 tokens
  Refined: Handle the auth flow.
```
